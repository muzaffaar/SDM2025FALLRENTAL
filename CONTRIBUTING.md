# Contributing to SDM Group Work

Thank you for considering contributing! Your help makes this project better for everyone.

---

## How to Contribute

1. **Fork the repository**  
   Click the fork button on GitHub to create your own copy.

2. **Clone your fork locally**  
   ```bash
   git clone https://github.com/your-username/SDM-Group-Work.git
   cd SDM-Group-Work
   ```

3. **Create a branch for your work**  
   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **Make your changes**  
   - Follow the existing code style.
   - Write clear, concise commits.
   - Update documentation if necessary.

5. **Run tests**  
   Ensure all unit and smoke tests pass.  
   ```bash
   ./vendor/bin/phpunit
   ```

6. **Push your branch**  
   ```bash
   git push origin feature/your-feature-name
   ```

7. **Open a Pull Request**  
   - Describe what your change does.  
   - Reference any related issues.  
   - Ensure your PR passes CI checks.

---

## Code Style

- **PHP:** PSR-12
- **JavaScript/React:** Airbnb style
- **Indentation:** 4 spaces in PHP, 2 spaces in JS
- Follow existing naming conventions and project structure.

---

## Reporting Issues

- Use the [GitHub Issues](https://github.com/your-org/SDM-Group-Work/issues) tracker.
- Provide clear steps to reproduce bugs.
- Include screenshots or logs if possible.

---

## Guidelines

- Be respectful and professional in comments and commits.
- Focus on small, incremental improvements.
- Make sure any external dependencies are properly documented.
- When in doubt, ask before making major changes.

---
